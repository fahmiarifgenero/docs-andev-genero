## Tampilan

Halaman depan ini menampilkan timeline dari suatu produk dengan menggunakan [layout front view](/layout-front-view) dapat dilihat pada gambar dibawah ini

![Halaman Depan](/front-view/development-journey/finish-good-1.png)

### List Product

<Callout type="warning" emoji="⚠️">
  Penggunaan query produk dengan URL belum berjalan
</Callout>

List produk terdapat pada pojok kanan atas menggunakan `select` yang dapat di pilih sesuai dengan produk yang di inginkan. List produk tersebut di ambil dari database. Untuk query produk dapat menggunakan URL parameter `?product_id=1` atau `?product_name=nama_produk`.

Note : Produk hanya dapat ditambah dan dihapus oleh <mark>office</mark> dan <mark>admin</mark>.

![List Produk](/front-view/development-journey/finish-good-5.png)

![List Produk](/front-view/development-journey/finish-good-4.png)

### List parameter

Contoh pada produk Aza 20 Cream menampilkan 3 parameter yaitu `BPO`, `Adapalene` dan `Nipagin`. Parameter di tiap - tiap produk bisa berbeda - beda, ada yang berjumlah 1, 2 atau 3. Pada contoh yang diambil terdapat 3 parameter. Data tersebut di ambil dari database. Jika query dibutuhkan dapat menggunakan URL parameter `?parameter_id=1`atau`?parameter_name=nama_parameter`.

Note : Produk hanya dapat ditambah dan dihapus oleh <mark>office</mark> dan <mark>admin</mark>.

![Parameter Produk](/front-view/development-journey/finish-good-6.png)

### Timeline Product

Timeline urut dari atas kebawah yaitu lebih baru ke lebih lama. Pada bagian kiri timeline terdapat waktu menunjukkan `timestamp` saat pertama kali data dibuat. Pada bagian kanan dari titik terdapat judul dan dibawah terdapat deskripsi singkat dari judul tersebut. Pada bagian kanan terdapat `badge` yang menunjukkan status dari data tersebut. Terdapat 4 status yaitu :

1. `kaji resource`
2. `pending`
3. `work in progress`
4. `done`

Status tersebut dapat di ubah hanya oleh `office` dan `admin`. Timeline ini dalam 1 parameter, untuk isi parameter lainnya serupa dengan ini.

## Halaman detail

<Callout type="warning" emoji="⚠️">
  Penggunaan query detail produk dengan URL belum berjalan
</Callout>

Untuk mengarahkan ke halaman detail dibutuhkan query pada URL contoh: `?id=1`. Pada halaman detail menampilkan detail dari satu timeline, misal ambil `id` ke 1 dari timeline parameter yaitu `BPO` pada produk `Aza 20 Cream`.

![Pilih halaman detail](/front-view/development-journey/finish-good-detail-2.png)

Selanjutnya user akan diarahkan ke bagian detail dari timeline tersebut. Berikut tampilan pada halaman detail.

![Halaman detail](/front-view/development-journey/finish-good-detail-1.png)

### Breadcrumb

Pada halaman detail di atas terdapat breadcrumb yang menunjukkan posisi user saat ini. Breadcrumb tersebut dapat di klik untuk kembali ke halaman sebelumnya. Breadcrumb tersebut di ambil dari URL parameter `?product_id=1` dan `?parameter_id=1`.

![Breadcrumb](/front-view/development-journey/finish-good-detail-3.png)

### Detail deskripsi

Pada bagian detail deskripsi terdapat judul dan deskripsi dari timeline tersebut. Status tersebut dapat di ubah hanya oleh `office` dan `admin`. Deskripsi berjumlah 5 accordion jika data tidak diisi maka accordion tersebut tidak akan muncul atau bertuliskan `NA`. Accordion diambil dari database dan dapat di tambahkan atau dihapus hanya oleh `office` dan `admin`.

![Detail deskripsi](/front-view/development-journey/finish-good-detail-5.png)

### Data detail

Data detail menampilkan data yang di input oleh `office`. Data tersebut dapat di tambahkan atau dihapus hanya oleh `office`. Data tersebut di ambil dari database.
Berikut tampilan data detail.

![Data detail](/front-view/development-journey/finish-good-detail-4.png)

form input yang digunakan pada data detail sebagai berikut :

| Nama Item    | Deskripsi                                                                                                                                                                     |
| ------------ | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| No Urut      | Berupa `number` dimulai dari 1 dan akan selalu bertambah jika data ditambahkan                                                                                                |
| Nama Dokumen | Berupa `string` dan tidak memerlukan editor                                                                                                                                   |
| Progress     | Input berupa `number` yang bisa dimasukkan dengan angka 1 - 100, dengan memiliki fitur berubah - ubah warna sesuai nilai yang dimasukkan dapat dilihat pada tab dibawah tabel |
| Keterangan   | Berupa deskripsi singkat dan membutuhkan editor dengan fitur `WYSIWYG`                                                                                                        |
| Download     | Berupa `button` untuk mendownload file yang telah di upload                                                                                                                   |
